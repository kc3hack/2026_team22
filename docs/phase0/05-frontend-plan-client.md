# 05: フロント - プラン API クライアント

## 概要

バックエンドのプラン取得 API を呼び出すための型定義と API クライアントを用意する。

## 完了条件

- [ ] プラン取得リクエストの型（カレンダー予定・睡眠ログ・設定）を定義している
- [ ] プラン取得レスポンスの型（週間睡眠プランの形）を定義している
- [ ] 04 のエンドポイントに POST で送り、プランを受け取る関数（またはフックの下層）がある
- [ ] 呼び出し時に 00 で用意した認証情報（Authorization ヘッダー等）を付与する
- [ ] 配置は `src/features/home/api/` または `src/features/home/` 内の `types.ts` + API 用ファイル（ARCHITECTURE に従う）

## メモ

- バックエンドのスキーマと整合させる（04 確定後）
- `API_URL` 等は既存の環境変数・定数を流用する
- 認証トークンは authStore や SecureStore から取得する
