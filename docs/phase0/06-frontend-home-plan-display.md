# 06: フロント - ホーム画面でプラン表示

## 概要

ホーム画面表示時にプラン取得 API を呼び出し、返ってきた週間睡眠プランを表示する。

## 完了条件

- [ ] ホーム画面（またはその子コンポーネント）で、表示時に 05 の API クライアントを呼び出している
- [ ] 呼び出し時に「カレンダー予定＋睡眠ログ＋設定」を渡している（取得方法はモック／既存機能でよい）
- [ ] 取得したプランを画面に表示している（リストやカードなど、最低限の UI でよい）
- [ ] ローディング・エラー状態を考慮している（任意: リトライ等）

## メモ

- `HomeScreen` は現状 `LightSensorScreen` を表示しているため、タブやセクションで「プラン」を追加する形でもよい
- カレンダー・睡眠ログ・設定の実データがまだない場合は、モックデータで API 連携まで確認できるようにする
- 未ログイン時はプラン API を呼ばない、または 00 の認証フローに誘導する
