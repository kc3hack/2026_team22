# 02: 署名ハッシュ生成

## 概要

「カレンダー予定＋睡眠ログ＋設定」を正規化し、その内容から Signature（ハッシュ値）を生成する。同じ入力なら同じハッシュになり、キャッシュヒット判定に使う。

## 完了条件

- [x] 入力データ（カレンダー予定・睡眠ログ・設定）を一意にシリアライズするロジックがある（例: キーソートした JSON）
- [x] その文字列からハッシュ（例: SHA-256）を生成する関数がある
- [x] バックエンドの `app/utils/signature.py` に配置され、03 の UseCase から呼び出せる

## メモ

- 順序に依存しないようにする（キーソートなど）
- 日付・時刻のフォーマットを揃える（ISO8601 など）
